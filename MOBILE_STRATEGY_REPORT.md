## 1. 대표님이 느낀 문제의 진짜 원인 (데이터로 진단)

대표님이 "PC는 100점인데 모바일은 50점도 안 될 것 같다"고 하신 느낌, 정확한 진단입니다.
코드를 스캔해서 **정량적 근거**를 찾았습니다.

| 항목 | 수치 | 의미 |
|---|---|---|
| 홈페이지 반응형 분기(`sm:`) | 75개 | 크기 조정은 잘 되어 있음 |
| 홈페이지 **모바일 정보 숨김**(`hidden sm:`) | **0개** | ⚠️ **모바일에서도 PC의 모든 정보를 100% 그대로 노출** |
| 홈 섹션 수 | 7개 (히어로/가치/신뢰배지/결과예시/가격/FAQ/CTA) | 세로로 매우 김 |

**결론:** 지금 모바일은 "PC 화면을 그대로 좁은 폭에 욱여넣은" 상태입니다.
글자·여백 크기는 줄였지만 **정보의 양(밀도)은 PC와 100% 동일**합니다.
좁은 화면에 같은 양의 정보 → **"조잡하고 복잡하다"는 인상**의 근본 원인입니다.

## 2. 성공한 서비스들은 어떻게 하는가 (벤치마킹)

### 토스 (Toss)
- 모바일에서 **한 화면에 한 가지 메시지**만. 스크롤하면 다음 메시지.
- 상세 수치·조건은 "자세히 보기"로 **접어둠**.
- 핵심: **덜어내기(subtraction)**. 정보를 줄이는 게 아니라 *순서대로 하나씩* 보여줌.

### 당근마켓
- 모바일에서 카드는 **핵심 3줄**(제목·가격·위치)만. 나머지는 탭하면 상세.
- 리스트는 무한스크롤이 아니라 **"청크(덩어리)"**로 끊어 보여줌.

### 대표적 SaaS 랜딩(Stripe, Linear, Notion)
- 데스크톱: 좌우 2~3단 레이아웃 + 풍부한 설명
- 모바일: **1단으로 접고**, 부가 설명은 **"더 알아보기"** 링크로 이동
- CTA(행동 유도 버튼)를 **화면 하단에 고정(sticky)** → 스크롤해도 항상 "무료 진단" 버튼이 보임

### 공통 법칙 3가지
1. **F-패턴/Z-패턴이 아니라 "1열 세로 흐름"** — 모바일은 무조건 세로 한 줄
2. **핵심 먼저, 상세는 접기** — 첫 화면(스크롤 없이 보이는 영역)에 가장 중요한 것만
3. **CTA는 항상 손 닿는 곳에** — 하단 고정 버튼

## 3. 우리 사이트 적용 전략 (A안 = 대표님 선택 · PC 불변)

> 대표님 지침: "자세한 내용은 PC에서 확인하라고 하면 관심 있는 고객은 PC로 넘어간다" ← 100% 동의.
> 전환 관점에서도, 모바일은 **'맛보기 + 신뢰 + 진단 유도'**, PC는 **'풍부한 정보 정독'** 역할 분담이 맞습니다.

### 홈페이지
- **히어로**: 모바일은 배지·부가문구 최소화, "무료 진단" 버튼을 크고 명확하게
- **신뢰 배지 7칸**: 모바일은 핵심 3~4개만 노출 + "그 외 강점 더보기"로 접기
- **AI 결과 예시본(노트북 목업)**: 모바일에서 가장 무거운 섹션.
  → 모바일은 **요약 카드 1~2개만** 보여주고 "실제 결과 화면 전체는 PC에서" 안내
- **FAQ 7개**: 모바일은 아코디언 유지(이미 접힘) — 양호
- **CTA**: 모바일 하단 **sticky 버튼** 추가 검토

### 진단 페이지 (diagnosis)
- 질문이 많음 → 모바일은 **한 번에 다 보이면 압도됨**
- 그룹별 간격·글자 최적화 + 진행률 표시 강화

### 결과 페이지 (matching-preview)
- 이미 아코디언(접기) 구조라 양호 → 여백·글자만 추가 정리

## 4. 구현 원칙 (안전 장치)

1. **PC 절대 불변**: 모든 변경은 `기본 클래스`(모바일)에만.
   PC 크기는 `sm:` 이상 클래스로 이미 고정 → 손대지 않음.
2. **정보 삭제 아님, "접기"**: 모바일에서 숨긴 정보도 "더보기"로 다 볼 수 있음.
   + "더 자세한 내용은 PC 버전에서 확인하세요" 부드러운 안내.
3. **디자인 훼손 없음**: 색상·폰트·카드 스타일 그대로. 정보 밀도/노출량만 조정.
